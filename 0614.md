电池退化模型已实现，包含后端算法、API 和前端「电池退化」页面。重启后端后即可使用（需先 pip install numpy）。

模型实现
1. SOC 估计（ECM + EKF）
一阶 RC 等效电路：V = OCV(SOC) - I·R0 - V1
EKF 状态：[SOC, V1]，用电流、端电压在线估计 SOC
OCV 曲线：磷酸铁锂商用包查表插值
2. SOH 半经验退化模型 L = f(DOD, SOC, T)
循环老化：

ΔSOH_cycle = k_c · DOD^β · exp(-Ea/R · (1/T - 1/T_ref)) · g(SOC)
g(SOC) 反映高/低 SOC 下的额外应力。

日历老化：

ΔSOH_cal = k_cal · exp(-Ea_cal/R · (1/T - 1/T_ref)) · t^0.5 · h(SOC)
RUL： 剩余等效满循环次数（EFC），EOL 阈值 SOH = 80%。

3. 退化成本
C_deg = ΔSOH × C_battery  （默认电池成本 80,000 元）

路径优化模块已实现，覆盖能耗模型、EVRPTW 求解、综合成本计算和前端可视化。

核心实现
1. 能耗模型 (energy/consumption.py)
基于车辆动力学：

F = F_r + F_g + F_a
P = Fv / η
能耗(kWh) = P × 行驶时间
2. EVRPTW 求解器 (routing/vrp_solver.py)
约束：时间窗、SOC 下限、充电站补能
算法：
insertion_2opt：最优插入法 + 2-opt 局部搜索
alns：ALNS 破坏-修复 + 2-opt
充电策略：SOC 不足时贪心插入最近充电站，充至 85%
3. 综合成本（已接入电池退化模型）
C_total = C_travel + C_energy + C_degradation
运输成本：距离 × 2 元/km
能耗成本：电量 × 0.8 元/kWh
退化成本：各路段 DOD 累加，调用电池退化模型

实现内容
1. 充电调度核心 (charge_scheduler.py)
目标：min(C_charge + C_waiting)
C_charge = Σ(充电量 kWh × 站点电价)
C_waiting = Σ(时间窗等待 + 排队等待) × 等待成本系数
输出 ChargePlan：充电站、充电量、充电时间、分项成本
支持在固定路径上优化充电量（在最低需求 SOC 与目标 SOC 之间搜索）
2. 与路径规划衔接 (path_planner.py)
综合成本现为完整公式：

C_total = C_travel + C_energy + C_charge + C_waiting + C_degradation

路径求解后自动调用充电调度，结果写入 charge_plan 与 cost_breakdown。

3. VRP 改进 (vrp_solver.py)
构造初始路线时先满足时间窗，再插入充电站满足 SOC
